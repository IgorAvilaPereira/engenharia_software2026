# Ciclo completo da Engenharia de Requisitos

## 1️⃣ Identificação de stakeholders

Primeiro é preciso identificar **quem tem interesse no sistema**.

Exemplos:

* usuários
* gestores
* clientes
* equipe técnica

---

## 2️⃣ Levantamento / elicitação de requisitos

Coletar informações sobre o sistema.

Métodos comuns:

* entrevistas
* questionários
* observação
* workshops
* prototipação

---

## 3️⃣ Análise de requisitos

Os requisitos coletados são analisados para:

* remover ambiguidades
* resolver conflitos
* entender dependências

---

## 4️⃣ Modelagem de requisitos

Representar os requisitos de forma estruturada.

Podem ser usados:

* diagramas de casos de uso
* diagramas UML
* fluxos de processo
* protótipos

---

## 5️⃣ Classificação de requisitos

Organizar os requisitos em categorias:

* **Requisitos Funcionais**
* **Requisitos Não Funcionais**

Também podem aparecer:

* requisitos de usuário
* requisitos de sistema
* regras de negócio

---

## 6️⃣ Priorização de requisitos

Definir **o que é mais importante implementar primeiro**.

Exemplo de método:

* **Método MoSCoW**

---

## 7️⃣ Especificação de requisitos

Documentar tudo formalmente no:

* **Documento de Especificação de Requisitos de Software**

Esse documento descreve:

* requisitos funcionais
* requisitos não funcionais
* interfaces
* regras de negócio
* casos de uso

---

## 8️⃣ Validação de requisitos

Verificar se os requisitos:

* representam corretamente o que o cliente quer
* estão completos
* não possuem erros

Pode envolver:

* revisões
* protótipos
* reuniões

---

## 9️⃣ Gerenciamento de requisitos

Controlar **mudanças durante o projeto**.

Inclui:

* rastreabilidade de requisitos
* controle de versões
* análise de impacto

---

# Fluxo simplificado

```id="cyv50f"
Stakeholders
      ↓
Levantamento
      ↓
Análise
      ↓
Modelagem
      ↓
Classificação
      ↓
Priorização
      ↓
Especificação (ERS)
      ↓
Validação
      ↓
Gerenciamento
```

---

✅ **Resumo importante**

As etapas **mais cobradas em provas** costumam ser apenas quatro:

* **elicitação**
* **análise**
* **especificação**
* **validação**

O restante são **detalhamentos dessas fases**.
