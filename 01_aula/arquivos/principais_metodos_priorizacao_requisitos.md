Além do **Método MoSCoW**, existem vários **métodos de priorização de requisitos** usados na **Engenharia de Software** para decidir **quais requisitos devem ser implementados primeiro** no **Documento de Especificação de Requisitos de Software**.

Aqui estão os principais:

---

# 1️⃣ Kano Model

Classifica os requisitos de acordo com **o impacto na satisfação do usuário**.

### Categorias

* **Básicos (Must-be)** → obrigatórios
* **Desempenho (Performance)** → quanto melhor, mais satisfação
* **Encantadores (Delighters)** → surpreendem o usuário

📌 Muito usado em **design de produto e UX**.

---

# 2️⃣ Analytic Hierarchy Process (AHP)

Método matemático para comparar requisitos **par a par**.

### Como funciona

Cada requisito é comparado com outro:

* requisito A é mais importante que B?
* quanto mais importante?

Depois é calculado um **peso de prioridade**.

📌 Muito usado em **projetos complexos**.

---

# 3️⃣ 100 Dollar Method

Também chamado de **Cumulative Voting**.

### Como funciona

Cada stakeholder recebe **100 pontos ou “dólares”** para distribuir entre os requisitos.

Exemplo:

| Requisito  | Pontos |
| ---------- | ------ |
| Login      | 40     |
| Relatórios | 35     |
| Dashboard  | 25     |

📌 Quanto mais pontos, maior a prioridade.

---

# 4️⃣ Value vs Effort Matrix

Classifica requisitos de acordo com:

* **valor para o negócio**
* **esforço de implementação**

Forma uma matriz com quatro quadrantes:

| Valor | Esforço | Prioridade     |
| ----- | ------- | -------------- |
| Alto  | Baixo   | Fazer primeiro |
| Alto  | Alto    | Planejar       |
| Baixo | Baixo   | Opcional       |
| Baixo | Alto    | Evitar         |

---

# 5️⃣ WSJF (Weighted Shortest Job First)

Muito usado em métodos ágeis.

### Fórmula

```
Prioridade = Valor do negócio / Tempo de desenvolvimento
```

📌 Requisitos com **mais valor e menor esforço** vêm primeiro.

---

# 📊 Comparação rápida

| Método          | Ideia principal                        |
| --------------- | -------------------------------------- |
| MoSCoW          | classificar por importância            |
| Kano            | impacto na satisfação do usuário       |
| AHP             | comparação matemática entre requisitos |
| 100 Dollar      | votação com pontos                     |
| Value vs Effort | valor vs esforço                       |
| WSJF            | valor dividido pelo tempo              |
