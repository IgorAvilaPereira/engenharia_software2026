Na **Engenharia de Software**, os requisitos mais conhecidos são os **funcionais** e **não funcionais**, mas existem **outras classificações** que também podem aparecer no **Documento de Especificação de Requisitos de Software**.

---

# 1️⃣ Requisitos de Negócio

Descrevem **os objetivos da organização** com o sistema.

📌 Respondem à pergunta: **por que o sistema existe?**

**Exemplo**

* O sistema deve permitir controle de vendas para melhorar o acompanhamento financeiro da empresa.

Eles definem **o propósito do sistema**, não as funções específicas.

---

# 2️⃣ Requisitos de Usuário

Descrevem **o que o usuário precisa fazer com o sistema**, geralmente de forma mais simples e menos técnica.

Normalmente aparecem como:

* **descrições textuais**
* **casos de uso**

**Exemplo**

* O usuário deve conseguir registrar um pedido.

---

# 3️⃣ Requisitos de Sistema

São **mais detalhados e técnicos**.

Eles descrevem exatamente **como o sistema deve funcionar** para atender os requisitos de usuário.

Exemplo:

* O sistema deve armazenar pedidos em banco de dados.
* O sistema deve validar login usando senha criptografada.

---

# 4️⃣ Regras de Negócio

São **regras que definem como o negócio funciona**.

Elas influenciam os requisitos do sistema.

**Exemplo**

* Um cliente só pode cancelar um pedido até 24 horas após a compra.
* Um aluno precisa ter presença mínima de 75%.

---

# 5️⃣ Requisitos de Interface

Descrevem **como o sistema interage com outras coisas**.

Podem incluir:

* interface com usuários
* interface com hardware
* interface com outros sistemas
* APIs

---

# 6️⃣ Requisitos de Dados

Descrevem **informações que o sistema deve armazenar ou manipular**.

Exemplo:

* O sistema deve armazenar nome, CPF e e-mail do cliente.

---

# 📊 Classificação comum de requisitos

| Tipo                      | Descrição                          |
| ------------------------- | ---------------------------------- |
| Requisitos de negócio     | objetivos da organização           |
| Requisitos de usuário     | necessidades dos usuários          |
| Requisitos de sistema     | especificação técnica do sistema   |
| Requisitos funcionais     | funções que o sistema executa      |
| Requisitos não funcionais | qualidades do sistema              |
| Regras de negócio         | regras que controlam processos     |
| Requisitos de interface   | interação com usuários ou sistemas |
| Requisitos de dados       | informações manipuladas            |

---

✅ **Resumo importante**

Hierarquia comum em engenharia de software:

```
Requisitos de Negócio
        ↓
Requisitos de Usuário
        ↓
Requisitos de Sistema
        ↓
Requisitos Funcionais e Não Funcionais
```
