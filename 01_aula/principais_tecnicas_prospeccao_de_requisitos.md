# Principais métodos de prospecção de requisitos

## 1️⃣ Entrevistas

Um dos métodos mais usados.

**Como funciona**

* analista conversa diretamente com usuários ou stakeholders
* faz perguntas sobre o sistema desejado

**Vantagens**

* informação detalhada
* permite esclarecer dúvidas

**Desvantagens**

* pode ser demorado
* depende da comunicação do entrevistado

---

## 2️⃣ Questionários

Usado quando há **muitos usuários**.

**Como funciona**

* formulário com perguntas
* enviado para vários usuários

**Vantagens**

* rápido para coletar dados
* alcança muitas pessoas

**Desvantagens**

* respostas menos detalhadas

---

## 3️⃣ Observação (Shadowing)

O analista **observa o usuário trabalhando**.

**Objetivo**

* entender como o processo realmente funciona.

**Vantagem**

* revela requisitos que usuários **nem percebem que existem**.

---

## 4️⃣ Workshops / reuniões de levantamento

Reuniões estruturadas com stakeholders.

Exemplo: sessões **JAD** (Joint Application Development).

**Vantagem**

* discussão coletiva
* decisões mais rápidas

---

## 5️⃣ Prototipação

Criar **protótipos do sistema** para os usuários testarem.

Relacionado ao **Prototyping Model**.

**Vantagens**

* usuários entendem melhor o sistema
* facilita descobrir requisitos

---

## 6️⃣ Análise de documentos

O analista analisa documentos existentes.

Exemplos:

* relatórios
* formulários
* sistemas antigos

**Vantagem**

* ajuda a entender regras de negócio.

---

## 7️⃣ Brainstorming

Sessão de geração de ideias com a equipe.

Objetivo:

* levantar **novas funcionalidades possíveis**.

---

# Comparação simples

| Método                | Objetivo principal               |
| --------------------- | -------------------------------- |
| Entrevistas           | obter requisitos detalhados      |
| Questionários         | coletar dados de muitos usuários |
| Observação            | entender processos reais         |
| Workshops/JAD         | discutir requisitos em grupo     |
| Prototipação          | validar ideias de sistema        |
| Análise de documentos | entender regras existentes       |
| Brainstorming         | gerar novas ideias               |

---

✅ **Resumo fácil de lembrar (muito usado em prova):**

Principais técnicas de prospecção:

* entrevistas

* workshops
* prototipação
* análise de documentos
