# Exemplo Modelo Kano

O **Modelo Kano** é uma técnica de **classificação de requisitos baseada na satisfação do usuário**. Ele foi criado por Noriaki Kano e busca entender **como diferentes funcionalidades impactam a percepção do usuário**.

Os requisitos são classificados em categorias principais:

* **Básicos (Must-be)** – esperados pelo usuário
* **Desempenho (Performance)** – quanto melhor, mais satisfação
* **Encantadores (Delighters)** – surpreendem o usuário
* **Indiferentes** – não afetam a satisfação
* **Reversos** – podem gerar insatisfação

---

# 1️⃣ Exemplo: Aplicativo de Delivery

### 🟥 Requisitos Básicos (Must-be)

Se não existirem, o usuário fica **muito insatisfeito**, mas quando existem **não geram grande satisfação**.

* Cadastro de usuário
* Carrinho de compras
* Pagamento online
* Confirmação do pedido

👉 O usuário **espera que isso funcione**.

---

### 🟨 Requisitos de Desempenho (Performance)

Quanto **melhor forem**, maior a satisfação.

* Tempo de entrega rápido
* Interface simples para escolher produtos
* Rastreamento do pedido em tempo real

👉 Melhor desempenho = **maior satisfação**.

---

### 🟩 Requisitos Encantadores (Delighters)

São **inesperados**, mas quando existem **encantam o usuário**.

* Sugestão automática de pratos baseada em preferências
* Cupons surpresa de desconto
* Notificação quando o entregador está chegando

👉 Se não existir, o usuário **não reclama**.

---

### ⚪ Requisitos Indiferentes

Não impactam muito a experiência.

* Troca de cores da interface
* Animações decorativas
* Sons no aplicativo

---

### ⚫ Requisitos Reversos

Alguns usuários podem **não gostar**.

* Notificações excessivas
* Reprodução automática de vídeos no app
* Compartilhamento automático em redes sociais

---

# 2️⃣ Exemplo: Sistema Acadêmico

### Básicos

* Matrícula em disciplinas
* Consulta de notas
* Consulta de horários

### Desempenho

* Rapidez para acessar notas
* Sistema estável em períodos de matrícula

### Encantadores

* Sugestão automática de disciplinas
* Visualização do progresso no curso (percentual concluído)

### Indiferentes

* Avatar personalizado do aluno

### Reversos

* Notificações frequentes demais
